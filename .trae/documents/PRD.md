## 1. 产品概述

### 1.1 产品名称
**ITOps Agent Platform** — 企业级 IT 运维多 Agent 自动化平台

### 1.2 核心价值主张
通过 AI 驱动的多 Agent 协作、可视化工作流编排、MCP 标准化工具协议，将告警处理、故障诊断、日志分析、容器管理、网络监控、数据中心运维等全场景运维任务自动化，实现对服务器、网络设备、容器、虚拟机、Kubernetes 集群、数据中心基础设施的统一智能管理。

### 1.3 目标用户
- 运维工程师 (SRE/DevOps)
- 系统管理员
- 网络工程师
- 数据中心运维人员
- IT 运维团队

### 1.4 产品范围

## 2. AI 智能体模块 (`ai/`)

### 2.1 Agent 管理

#### Agent 实体定义

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 唯一标识符 |
| name | String | Agent 名称（如"告警处理Agent"） |
| avatar | String | 头像 emoji 或图标 |
| role | String | 角色描述 |
| systemPrompt | Text | 系统提示词 |
| model | Enum | 模型选择（doubao-4o / gpt-4o / gpt-4-turbo） |
| temperature | Float | 温度参数 0.0-2.0 |
| enabled | Boolean | 是否启用 |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

#### 预设 Agent 模板（7 个）

| Agent | Emoji | 角色 |
|-------|-------|------|
| 告警处理 Agent | 🚨 | 接收和初步分析告警信息，判断严重程度 |
| 故障诊断 Agent | 🔍 | 分析故障原因，生成排查步骤清单 |
| 日志分析 Agent | 📊 | 解析日志格式，识别异常模式 |
| 系统巡检 Agent | 🖥️ | 检查服务器 CPU/内存/磁盘/网络状态 |
| 变更执行 Agent | 🔧 | 安全执行系统命令和变更操作 |
| 文档生成 Agent | 📝 | 生成故障报告、巡检报告、变更记录 |
| 合规检查 Agent | ✅ | 检查配置合规性，生成合规报告 |

#### Agent 管理功能
- 创建/编辑/删除自定义 Agent
- 批量删除
- 导入/导出 Agent 配置（JSON 格式）
- 启用/禁用 Agent
- Agent 执行历史查询

### 2.2 多 Agent 协作系统

- **Coordinator（协调器）**: 接收任务，按规则分发给 Specialist
- **Specialist 专家**: 7 个预设专家 + 可扩展注册
- **协作流程**: Coordinator → Specialist Registry → 匹配专家 → 并行/串行执行 → 结果聚合
- 支持执行流程可视化和日志追踪

### 2.3 LLM Provider 管理

- **Provider 注册**: 支持豆包、OpenAI，可扩展 Anthropic / 本地模型
- **API 模型管理**: 模型 CRUD、API Key 配置、Base URL 配置
- **熔断机制 (Circuit Breaker)**: 连续失败 5 次自动降级到备用 Provider
- **Provider 健康检查**: 定期探活，自动切换

### 2.4 RCA 根因分析

- **告警触发 RCA**: 接收告警信息，自动启动根因分析流程
- **本地规则引擎**: 基于预定义规则的快速诊断
- **LLM 深度分析**: 调用 AI 模型进行复杂根因推理
- **分析记录**: RCA 历史记录查询、详情查看
- **关联推荐**: 基于历史 RCA 案例推荐解决方案

### 2.5 AI 修复建议

- **智能修复推荐**: 基于告警内容和知识库生成修复方案
- **增强 RAG 服务**: 结合向量检索和 LLM 生成精准修复步骤
- **修复工作台**: AI 修复建议的查看、采纳、执行追踪
- **知识反馈闭环**: 修复结果反馈到知识库，持续优化

### 2.6 AI 知识库

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 唯一标识符 |
| title | String | 标题 |
| category | String | 分类（故障案例/最佳实践/操作手册） |
| tags | Array | 标签 |
| content | Text | 内容 |
| solutions | JSON Array | 解决方案数组 |
| relatedAlerts | Array | 相关告警类型 |
| usageCount | Integer | 使用次数 |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

#### 功能
- 知识条目 CRUD
- 全文搜索 + 标签筛选
- 自动关联告警类型
- QAnything 智能问答
- 知识推荐（基于告警内容）
- 使用次数统计

### 2.7 AI 对话助手

- **ChatWidget**: 全局 AI 对话浮窗
- **上下文感知**: 根据当前页面提供相关建议
- **推荐卡片**: 在关键页面展示 AI 推荐操作

## 3. 告警管理模块 (`alerts/`)

### 3.1 告警实体

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 唯一标识符 |
| source | String | 告警来源（Prometheus/Grafana/Zabbix/自定义） |
| severity | Enum | 严重程度：critical/high/medium/low/info |
| title | String | 告警标题 |
| content | Text | 告警内容 |
| metadata | JSON | 告警元数据（IP、服务名等） |
| status | Enum | 状态：new/confirmed/in_progress/resolved/resolved_auto/ignored |
| createdAt | DateTime | 告警时间 |
| resolvedAt | DateTime | 解决时间 |
| relatedTaskId | UUID | 关联的任务 ID |

### 3.2 告警全生命周期

```
外部告警源 → AlertProvider 适配接收 → 告警过滤/降噪
    → 告警关联分组 → 自动分析（设备解析/模式识别）
    → 通知分发（邮件/Webhook/钉钉/企业微信）
    → 可选自动响应（巡检/诊断/修复）→ 关闭
```

### 3.3 告警列表与筛选
- 多维度筛选：严重程度、状态、时间范围、来源、关键词搜索
- 分页 + 排序（创建时间/严重程度）
- 批量操作：确认/忽略/解决
- 告警详情：完整内容、元数据、关联设备、关联任务

### 3.4 告警自动分析
- **设备自动解析**: 从告警内容中提取 IP/主机名/设备类型
- **模式识别**: 识别告警模式（突增/周期/持续）
- **影响分析**: 评估告警对业务的影响范围
- **智能通知**: 根据告警严重程度和用户偏好选择性通知

### 3.5 告警降噪管理
- **降噪规则**: 配置告警聚合/抑制/静默规则
- **去重策略**: 相同告警在时间窗口内合并
- **静默时段**: 维护窗口期自动静默
- **降噪统计**: 降噪效果可视化

### 3.6 告警关联分组
- **关联算法**: 基于时间/设备/标签/内容相似度的自动分组
- **关联组视图**: 分组展示关联告警，支持手动调整
- **根因推断**: 从关联组中识别主告警和衍生告警

### 3.7 告警 Provider 管理
- 内置适配：Prometheus、Grafana、Zabbix、Webhook 通用
- Provider CRUD：新增/编辑/删除告警源配置
- Webhook 日志：记录所有 Webhook 接收日志
- 自动响应配置：绑定告警类型到修复工作流

### 3.8 巡检中心
- **巡检任务管理**: 创建/编辑/删除巡检任务
- **巡检模板**: 预定义巡检项（CPU/内存/磁盘/服务/进程）
- **巡检历史**: 巡检记录查询，结果对比
- **SNMP 巡检**: 网络设备巡检结果展示

## 4. 认证授权模块 (`auth/`)

### 4.1 用户管理

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 唯一标识符 |
| username | String | 用户名 |
| email | String | 邮箱 |
| role | Enum | 角色：admin / operator / viewer |
| passwordChangedAt | DateTime | 上次改密时间 |
| createdAt | DateTime | 创建时间 |

### 4.2 三级角色权限

| 能力 | viewer（查看者） | operator（运维者） | admin（管理员） |
|------|:---:|:---:|:---:|
| 查看仪表盘/告警/监控 | ✅ | ✅ | ✅ |
| 管理服务器/网络设备 | ❌ | ✅ | ✅ |
| 执行 SSH 命令 | ❌ | ✅ | ✅ |
| ~~危险命令（rm/dd/kill）~~ | ❌ | ❌ | ✅（已删除 2026-07-06，无命令过滤）|
| ~~系统关键操作（mkfs/reboot）~~ | ❌ | ❌ | ❌（已删除 2026-07-06，无命令过滤）|
| 管理用户/角色 | ❌ | ❌ | ✅ |
| 系统设置 | ❌ | ❌ | ✅ |

### 4.3 安全机制
- **JWT 认证**: Token 过期自动刷新
- **密码策略**: 最小 8 位，复杂度要求
- **强制改密**: 首次登录/密码过期强制修改
- **登录限流**: 连续失败锁定
- **Token 黑名单**: 主动登出 Token 即时失效
- **密码加密**: bcryptjs 哈希存储
- **加密密钥管理**: 敏感数据（SSH 密码/API Key）加密存储

## 5. 自动化运维模块 (`auto/`)

### 5.1 修复策略管理

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 唯一标识符 |
| name | String | 策略名称 |
| description | Text | 策略描述 |
| triggerCondition | JSON | 触发条件（告警类型/严重程度/关键字） |
| remediationSteps | JSON Array | 修复步骤列表 |
| approvalRequired | Boolean | 是否需要审批 |
| enabled | Boolean | 是否启用 |
| createdAt | DateTime | 创建时间 |

#### 功能
- 修复策略 CRUD
- 策略编辑器：可视化配置触发条件和修复步骤
- 策略测试：模拟触发验证
- 绑定工作流：修复策略可关联工作流自动执行

### 5.2 修复执行管理
- **执行工作台**: 查看所有修复执行记录
- **执行详情**: 步骤级执行日志
- **实时进度**: WebSocket 推送执行状态
- **人工干预**: 暂停/继续/取消/跳过步骤
- **验证回滚**: 修复后自动验证，失败自动回滚

### 5.3 修复审计
- **审计追踪**: 记录每次修复的完整过程
- **成功率统计**: 按策略/设备/时间维度统计
- **审计报告**: 导出修复审计报告

### 5.4 自动伸缩
- **伸缩规则**: 基于 CPU/内存/流量指标的自动伸缩策略
- **伸缩历史**: 伸缩事件记录和趋势分析
- **规则管理**: CRUD + 启用/禁用

### 5.5 修复仪表盘
- 修复概览（总数/成功率/平均耗时）
- 策略执行趋势图
- 最近修复记录
- 待处理修复队列

## 6. 容器与虚拟机模块 (`containers/`)

- 日志查看器：实时日志流 + 历史日志搜索

- 多容器实时资源对比

- 镜像操作：拉取/删除/打标签

#### 镜像仓库
- 仓库配置 CRUD（URL、认证信息）
- 仓库镜像浏览
- 镜像推送/拉取

### 6.4 Docker Compose 管理
- **Compose 项目**: 上传/编辑/删除 docker-compose.yml

### 6.5 存储卷管理

- **卷操作**: 创建/删除/查看详情

#### 多平台支持

- VM 创建向导：选择平台 → 配置 CPU/内存/磁盘/网络 → 选择 ISO → 创建

#### VM 快照管理

- 快照列表：名称、时间、描述
- 快照恢复：回滚到指定快照
- 快照删除

#### 快照策略
- 定时快照策略：按天/周/月自动创建
- 保留策略：最多保留 N 个快照
- 策略管理：CRUD + 启用/禁用

#### VM 迁移
- 在线迁移：源主机 → 目标主机
- 迁移历史：记录每次迁移的源/目标/状态/耗时
- 迁移状态追踪

## 8. 数据中心模块 (`dc/`)

### 8.1 数据中心管理

#### 机房管理
- 机房 CRUD：名称、位置、面积、描述
- 机房概览：机架数量、设备数量、功耗统计

#### 机架管理
- 机架 CRUD：名称、位置（机房内）、高度（U 数）
- 槽位管理：每个 U 位的设备占用情况
- 机架面板视图：可视化 U 位占用（空/占用/预留）

#### 设备管理
- 设备 CRUD：名称、类型（服务器/交换机/路由器/防火墙/存储/PDU）
- 设备制造商管理（厂商 CRUD + 型号管理）
- 设备类型管理（槽位定义模板）
- 设备生命周期：上架/下架/维修/报废
- 设备详情：配置、关联机架槽位、关联线缆

#### PDU 管理
- PDU CRUD：名称、类型、额定功率
- 端口管理：每个端口的电源状态和负载
- PDU 端口关联设备

#### 电源管理
- 电源面板 CRUD：输入电压、额定容量
- 电源馈线 CRUD：从面板到设备的供电线路

#### 线缆管理
- 线缆 CRUD：类型（光纤/铜缆/电源线）、起点设备端口、终点设备端口

## 16. 非功能需求

### 16.1 性能需求
- 工作流执行响应时间 < 5s（不含 AI 调用）
- WebSocket 消息推送延迟 < 100ms
- 页面首次加载时间 < 3s（懒加载优化）
- 支持 100+ 并发任务执行
- 数据库 WAL 模式支持高并发读取

### 16.2 安全需求
- 所有 API 密钥后端管理，前端不可见
- HTTPS 加密传输
- JWT 认证 + 三级角色权限
- ~~SSH 命令 6 层安全过滤~~（已删除 2026-07-06）
- 输入输出 XSS 防护（DOMPurify）
- 操作审计日志完整记录
- 敏感数据加密存储（bcryptjs + 自定义加密）

### 16.3 可靠性需求
- 完善的错误处理机制（全局 errorHandler）
- 自动重试机制（SSH 连接、AI 调用、任务执行）
- 任务状态持久化（SQLite 即时写入）
- 数据备份恢复
- 服务优雅关闭（逆序 Shutdown）
- LLM Provider 熔断降级

### 16.4 可用性需求
