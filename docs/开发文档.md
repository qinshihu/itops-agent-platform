# AI 智能根因分析与一键自愈系统（AIOps 杀手锏功能）

## 一、产品定位

**核心价值：** 告警触发后，AI 自动完成"检测 → 定位根因 → 推荐修复 → 一键执行 → 效果验证"的全链路自动化，运维人员只需点击确认。

**对标竞品：** Datadog Watchdog ($5000+/年)、Dynatrace Davis AI ($3000+/年)、PagerDuty AIOps ($1200/年/人)

**差异化卖点：** 免费、开箱即用、中文优先、支持私有化部署

## 五、Phase 2: AI 根因分析引擎

### 5.1 分析流程

```
告警触发
  │
  ▼
Step 1: 收集上下文 (5 秒)
  ├─ 告警详情 (server_id, severity, message, metric)
  ├─ 受影响的服务拓扑 (上游/下游)
  ├─ 最近 24h 变更记录
  ├─ 最近 1h 同类告警
  ├─ 服务器当前状态 (SSH 检测)
  └─ 知识库相似案例
  │
  ▼
Step 2: LLM 根因分析 (15-30 秒)
  ├─ 构建 Prompt (结构化上下文)
  ├─ 调用 LLM (llmService)
  ├─ 解析输出
  └─ 规则引擎交叉验证
  │
  ▼
Step 3: 生成分析报告
  ├─ 根因定位 (哪个服务/配置/网络问题)
  ├─ 影响链路 (A → B → C)
  ├─ 置信度评分 (0-100%)
  ├─ 修复建议 (Top 3)
  └─ 知识库匹配方案
  │
  ▼
Step 4: 推送通知
  ├─ 创建 RCA 记录
  ├─ 关联告警
  ├─ 推送通知 (WebSocket/邮件)
  └─ 标记为"可自愈"(如果匹配到修复策略)
```

### 10.1 SSH 拓扑发现实现

```typescript
// 通过 SSH 扫描服务器端口发现依赖
async discoverDependencies(serverId: string): Promise<Dependency[]> {
  const server = await serverService.getById(serverId);
  const conn = await sshService.createConnection(server, serverId);

  // 1. 获取网络连接
  const netstat = await sshService.execCommand(conn, 'netstat -tunapl 2>/dev/null || ss -tunapl');
  const connections = parseNetstatOutput(netstat);

  // 2. 获取进程信息
  const processes = await sshService.execCommand(conn, 'ps aux');
  const serviceNames = parseServiceNames(processes);

  // 3. 获取配置文件
  const configFiles = await sshService.execCommand(conn, 'find /etc -name "*.conf" -o -name "*.yml" -o -name "*.yaml" 2>/dev/null');

  // 4. 匹配已知服务器 IP
  const knownServers = await serverService.getAll();
  const dependencies = matchKnownServers(connections, knownServers);

  return dependencies;
}
```

## 十一、安全考虑

| 风险       | 防护措施                        |
| :------- | :-------------------------- |
| SSH 密钥解密 | 内存中解密，不落盘，使用后立即释放           |
| 修复操作审计   | 所有修复操作记录审计日志 (谁、何时、做了什么、结果) |
| 高危操作审批   | 中高风险修复需要人工确认，不可自动执行         |
| 回滚能力     | 每个修复操作生成回滚命令，支持一键回滚         |
| LLM 输出验证 | 规则引擎交叉验证，防止 LLM 幻觉          |
| 权限控制     | 自愈操作需要 admin 或 operator 角色  |
