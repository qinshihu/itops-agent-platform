## 四、部署与启动脚本

### 4.1 一键部署脚本（拉取镜像 + 启动）

| 文件                        | 用途                                                                                                                                                                                                                                                                                               |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [deploy.sh](../deploy.sh)   | Linux/Mac 一键部署脚本。默认从阿里云镜像仓库（`registry.cn-hangzhou.aliyuncs.com/huluwa666/tsq-images-hub`）拉取 `IT_Onlin-ITOps-{backend\|frontend}-latest` 镜像，自动生成 `docker-compose.yml`，检查端口 3001/8080 占用，启动服务并验证健康检查。支持 `-d/--dir`、`-y/--yes`、`-u/--update` 参数 |
| [deploy.ps1](../deploy.ps1) | Windows PowerShell 一键部署脚本。功能同 `deploy.sh`，额外支持 `-JwtSecret`、`-BackendPort`、`-FrontendPort`、`-Version` 参数。会自动生成 `.env` 文件并智能增量更新（保留已有配置）                                                                                                                 |

### 4.2 镜像构建与推送脚本

| 文件                                              | 用途                                                                                                                                                                                      |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [docker-build-push.sh](../docker-build-push.sh)   | Linux/Mac 镜像构建推送脚本。使用 `docker/Dockerfile.backend` 和 `docker/Dockerfile.frontend` 构建镜像，支持 `-r/--registry`、`-u/--username`、`-v/--version`、`--push`、`--no-cache` 参数 |
| [docker-build-push.ps1](../docker-build-push.ps1) | Windows PowerShell 版镜像构建推送脚本，功能同上                                                                                                                                           |

### 4.3 开发环境启动方式

本地开发环境的启动方式已统一收敛为两种（详见 [.trae/rules/top-rules.md §四](../.trae/rules/top-rules.md)）：

1. **宿主机直接运行**（最快，推荐日常编码调试）：
   - `cd backend; npm run dev` → http://localhost:3001
   - `cd frontend; npm run dev` → http://localhost:3000
2. **local-dev Docker 开发环境**（环境与生产一致，推荐调试/演示）：见 [`local-dev/`](../local-dev) 目录。

> **历史变更**：2026-07-20 之前根目录存在 `start.sh`/`start.ps1`/`stop.sh`/`stop.ps1`/`start_backend.ps1` 等 Docker Compose 本地构建启动脚本，因与 `local-dev/` 目录功能重复且容易引起混淆，已被移除。

### 4.4 Docker Compose 配置

| 文件                                        | 用途                                                                                                                                                                                                                                                            |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [docker-compose.yml](../docker-compose.yml) | 生产环境 Docker Compose 配置。定义 `backend`（2 CPU/2G 内存）和 `frontend`（1 CPU/512M 内存）两个服务，挂载 `app-data` 和 `app-backups` 卷，挂载 Docker socket 支持容器管理，配置完整环境变量（JWT/LLM API/Webhook/Email/Allowed Origins），健康检查 + 日志轮转 |
