### P0-1：approval_requests 同名表 schema 冲突（沉默漂移）

**问题**：[v017_approval_requests.ts:11](file:///f:/自开发代码/多Agent运维平台/ITops%20agent/daima/backend/src/models/migrations/v017_approval_requests.ts#L11) 与 [v021_approval_requests.ts:5](file:///f:/自开发代码/多Agent运维平台/ITops%20agent/daima/backend/src/models/migrations/v021_approval_requests.ts#L5) 都建 `approval_requests` 表，字段完全不同：

- v017 字段：`task_id, node_id, node_label, requested_by, approved_by, approved_at, reject_reason, timeout_at, timeout_action` + FK tasks

   ```bash
   # 搜索 v021 字段使用
   grep -rn "request_type\|related_type\|related_id\|payload" backend/src --include="*.ts" | grep -i "approval"
   ```

2. **根据调研结果选择方案**：

   **方案 B（若 v021 是新版审批，v017 已废弃）**：

### P0-2：VNC 代理无鉴权 + SSRF

**问题**：[vncProxyService.ts:19](file:///f:/自开发代码/多Agent运维平台/ITops%20agent/daima/backend/src/modules/network/services/vncProxyService.ts#L19) 使用 `io.of('/vnc')` 独立命名空间，连接处理函数内**无任何鉴权**；L22 直接接受客户端传入的 `vncHost/vncPort/password`——构成 SSRF + 未授权访问风险（可让服务器向任意 IP:Port 发起 TCP 连接）。

**根因**：独立命名空间绕过 [shared/websocket/handler.ts:56](file:///f:/自开发代码/多Agent运维平台/ITops%20agent/daima/backend/src/modules/network/services/vncProxyService.ts) 的全局 `io.use(authenticateSocket)`。

**修复步骤**：

1. 在 [vncProxyService.ts](file:///f:/自开发代码/多Agent运维平台/ITops%20agent/daima/backend/src/modules/network/services/vncProxyService.ts) 的 `/vnc` 命名空间挂载鉴权中间件：
   ```typescript
   vncNamespace.use((socket, next) => {
     const token = socket.handshake.auth?.token || socket.handshake.headers?.authorization?.replace('Bearer ', '');
     if (!token) return next(new Error('UNAUTHORIZED'));
     // 复用 middleware/auth.ts 的 verifyJWT 逻辑
     verifyJWT(token).then(user => { socket.data.user = user; next(); }).catch(() => next(new Error('INVALID_TOKEN')));
   });
   ```

2. **SSRF 防护**：在连接前校验 `vncHost/vncPort`：
   - 维护允许的 VNC 目标白名单（从 `network_devices` 表查询，仅允许已注册设备的 management IP）
   - 拒绝私有保留地址（`127.0.0.0/8`、`169.254.0.0/16`、`0.0.0.0`）除非显式允许
   - 拒绝云元数据端点（`169.254.169.254`）

3. 确认 `initialize(io)` 是否被实际调用（grep `vncProxyService.initialize` 或 `vncProxy` 的引用），若未调用则当前是潜伏风险，但仍需修复。

**验收标准**：
- [ ] 未携带 token 的 `/vnc` 连接被拒绝
- [ ] 传入非白名单 IP 的连接被拒绝
- [ ] 新增鉴权单测覆盖

   ```typescript
   function isIpWhitelisted(ip: string, whitelist: string[]): boolean {

   ```
